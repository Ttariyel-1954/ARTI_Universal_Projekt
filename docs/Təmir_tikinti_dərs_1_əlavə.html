1 Giriş və domen analizi

Təmir-Tikinti İdarəsi (TTİ) — Azərbaycan Respublikası Təhsil Nazirliyi tabeliyində fəaliyyət göstərən idarədir. Əsas funksiyası: ölkə üzrə bütün məktəb, lisey və gimnaziyaların tikinti-təmir işlərini planlaşdırmaq, icra etmək və monitorinq aparmaqdır.

1.1. TTİ-nin əsas fəaliyyət sahələri

  • 📍 Regional infrastruktur — 77 rayon, 4,500+ məktəb
  • 🏗️ Layihə idarəetməsi — kapital təmir, əsaslı tikinti, cari təmir
  • 💰 Büdcə və maliyyə — illik 300+ milyon AZN büdcə
  • 📦 Maddi-texniki baza — 15+ anbar, binlərlə material növü
  • 👷 İşçi heyət — 500+ daxili işçi, 50+ podratçı şirkət
  • 📋 İcra monitorinqi — aktlar, inspeksiyalar, kvartal hesabatları

1.2. İdarəetmə çətinlikləri

!
Real problemlər
  • 77 rayondan eyni anda 120+ layihənin koordinasiyası
  • Büdcə icra faizi və xərclərin real-time monitorinqi
  • Materialların anbardan layihəyə paylanması
  • Podratçı şirkətlərin performans qiymətləndirilməsi
  • Aktların, imzaların, qəbulların izlənməsi
  • Vətəndaş müraciətlərinin və şikayətlərin cavablandırılması

1.3. Verilənlər bazası niyə lazımdır?

Əvvəllər bütün bu məlumatlar Excel cədvəllərində, kağız üzərində saxlanılırdı. Enterprise DB qurmaqla əldə ediləcəklər:

  • Bir sorğuda 77 rayon üzrə tam mənzərə
  • Real-time büdcə və icra statistikası
  • Avtomatik hesabat generasiyası
  • AI-əsaslı təhlil və proqnoz
  • Tarixi data saxlanması və trend analizi
  • Müxtəlif rollara (direktor, inspektor, mühasib) uyğun interfeys

13 Trigger-lər və constraint-lər

Bazanın özündə biznes qaydalarını saxlayırıq — bu, backend-də səhv kod olsa da, məlumatların düzgünlüyünü qoruyur.

13.1. Trigger — updated_at avtomatik

SQL — Updated_at trigger
CREATE OR REPLACE FUNCTION update_timestamp()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_projects_updated
  BEFORE UPDATE ON construction.projects
  FOR EACH ROW EXECUTE FUNCTION update_timestamp();

13.2. Trigger — budget sərf avtomatik hesablanması

SQL — Budget tracker trigger
CREATE OR REPLACE FUNCTION update_budget_spent()
RETURNS TRIGGER AS $$
BEGIN
  -- Yeni xərc əlavə olanda, müvafiq büdcənin spent_amount-u yenilənir
  UPDATE finance.budgets
  SET spent_amount = spent_amount + NEW.amount
  WHERE fiscal_year = EXTRACT(YEAR FROM NEW.expense_date)
    AND region_id = (SELECT region_id FROM construction.projects
                      WHERE id = NEW.project_id);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_expense_update_budget
  AFTER INSERT ON finance.expenses
  FOR EACH ROW EXECUTE FUNCTION update_budget_spent();

13.3. Trigger — stok qalığı avtomatik hesablanması

SQL — Stock trigger
CREATE OR REPLACE FUNCTION update_stock_level()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO inventory.stock_levels (warehouse_id, material_id, quantity, last_updated)
  VALUES (NEW.warehouse_id, NEW.material_id,
          CASE WHEN NEW.transaction_type = 'in'  THEN NEW.quantity
               WHEN NEW.transaction_type = 'out' THEN -NEW.quantity
          END, NOW())
  ON CONFLICT (warehouse_id, material_id) DO UPDATE
  SET quantity = stock_levels.quantity +
      CASE WHEN NEW.transaction_type = 'in'  THEN NEW.quantity
           WHEN NEW.transaction_type = 'out' THEN -NEW.quantity
      END,
      last_updated = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_stock_auto_update
  AFTER INSERT ON inventory.material_transactions
  FOR EACH ROW EXECUTE FUNCTION update_stock_level();

15 Materialized view-lar — sürət üçün

Artıq arti_schools-da gördük: materialized view SQL nəticəni diskdə saxlayır — 100-6000x sürət artımı.

SQL — Materialized views
-- Əsas KPI dashboard üçün
CREATE MATERIALIZED VIEW reporting.mv_kpi_dashboard AS
SELECT
  (SELECT COUNT(*) FROM geo.regions WHERE is_active)           AS active_regions,
  (SELECT COUNT(*) FROM infra.schools WHERE is_active)          AS active_schools,
  (SELECT COUNT(*) FROM construction.projects)                   AS total_projects,
  (SELECT COUNT(*) FROM construction.projects 
   WHERE status = 'in_progress')                                 AS active_projects,
  (SELECT ROUND(SUM(planned_budget), 0) FROM construction.projects) AS total_planned,
  (SELECT ROUND(SUM(actual_cost), 0) FROM construction.projects)   AS total_spent,
  (SELECT ROUND(AVG(progress_percent), 1) FROM construction.projects) AS avg_progress;

-- Avtomatik refresh üçün
CREATE OR REPLACE FUNCTION reporting.refresh_all_mv()
RETURNS TEXT AS $$
BEGIN
  REFRESH MATERIALIZED VIEW reporting.mv_kpi_dashboard;
  REFRESH MATERIALIZED VIEW reporting.mv_region_ranking;
  REFRESH MATERIALIZED VIEW reporting.mv_top_projects;
  REFRESH MATERIALIZED VIEW reporting.mv_monthly_progress;
  RETURN '✅ Bütün MV-lər yeniləndi';
END;
$$ LANGUAGE plpgsql;
